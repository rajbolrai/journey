Project set-up

1. Make 'mkdir directory_name' directory or 'cd' into the correct directory
2. Make virtual environment 'python3 -m venv venv_name'
3. Activate the virtual environment and add the relevant modules
4. Add the modules into a requirement txt file 'pip freeze > requirement.txt'
5. Create a .gitignore file and add virutal folder so git ignores it
6. Git init and create repository in Github and get url then link remote origin
7. Stage the project and commit then push it to main.

Python Data structures

Classes

Blueprint for object. Object can store data and replicate behaviour, thus, are used to represent model entities for example car object
Defined by class classname():
__init__(self) is used to initialise the class
The 'self' keyword must be the first argument of every method within the class.
It represent the current instance of the class. An instance will have access to all its public variables and methods.
__str__ : method to have in each class. A more descriptive way to represents the custom object
__repr__: method to have in each class. Concise way to represents the custom object

Multiple Classes

When creating mult-class system, rememeber to:
1. Use the ‘single responsibility’ principle for modules or classes.
2. Use the ‘open-closed’ principle for extending the functionality of the program.

Nested list and dictionary

Nested Dictionary is a dictonary where each key is the dictionary name and the value is a dictionary.
List of dictionary contains dictionary at each entry.

Testing

Unit tests are for testing functions or singular classes
Integration test is a test that tests multiple classes that are coupled together. It checks for their functionality.
Error handling: Remember to catch the correct error type. Makes it easier to organise and see.
Mocking is useful when you want to test the functionality of a module or class without relying on referenced classes. It allows you to isolate the module, so if anything goes wrong, then it is the module’s or classes’ fault, not that of the coupled classes.
Using mocks for API tests and to turn anything that is non-deterministic to a deterministic value

Relational Database

The choice of Relational Database for these projects is PostgreSQL.
Database tables consist of:
- columns: attributes of a row
- row: singular object like structure that houses varius data
Also tables can be linked to each other using a special column called keys, hence the name relational
Each tabe has a primary key that PostgreSQL assigns. A table can reference another table by storing the other's primary key in a column in what is known as 'foreign key'

Seeds are initial sets of data which can be used to populate a database and its tables

Querying data syntax:
SELECT [list of columns] FROM [table_name];

Updating a value:
UPDATE [table_name] SET [column_name] = [new_value];

Deleting a specific rows that match a certain condition
DELETE FROM [table_name] WHERE [conditions];

Adding new row
INSERT INTO [table_name]
( [list of columns] )
VALUES( [list of values] );

To turn the data obtained from PostgreSQL, use a ‘Model’ and ‘Repository’ classes (sometimes called ORM). Model class will hold a record’s data, for example, an attribute for each column. A repository class holds methods to run SQL queries on the database to retrieve, create, update or delete data.
