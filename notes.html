Project set-up

1. Make 'mkdir directory_name' directory or 'cd' into the correct directory
2. Make virtual environment 'python3 -m venv venv_name'
3. Activate the virtual environment and add the relevant modules
4. Add the modules into a requirement txt file 'pip freeze > requirement.txt'
5. Create a .gitignore file and add virutal folder so git ignores it
6. Git init and create repository in Github and get url then link remote origin
7. Stage the project and commit then push it to main.

Python Data structures

Classes

Blueprint for object. Object can store data and replicate behaviour, thus, are used to represent model entities for example car object
Defined by class classname():
__init__(self) is used to initialise the class
The 'self' keyword must be the first argument of every method within the class.
It represent the current instance of the class. An instance will have access to all its public variables and methods.
__str__ : method to have in each class. A more descriptive way to represents the custom object
__repr__: method to have in each class. Concise way to represents the custom object

Nested list and dictionary

Nested Dictionary is a dictonary where each key is the dictionary name and the value is a dictionary.
List of dictionary contains dictionary at each entry.

Testing

Unit tests are for testing functions or singular classes
Integration test is a test that tests multiple classes that are coupled together. It checks for their functionality.
Error handling: Remember to catch the correct error type. Makes it easier to organise and see.
