Testing

Unit tests are for testing functions or singular classes
Integration test is a test that tests multiple classes that are coupled together. It checks for their functionality.
Error handling: Remember to catch the correct error type. Makes it easier to organise and see.
Mocking is useful when you want to test the functionality of a module or class without relying on referenced classes. It allows you to isolate the module, so if anything goes wrong, then it is the module’s or classes’ fault, not that of the coupled classes.
Using mocks for API tests and to turn anything that is non-deterministic to a deterministic value

Relational Database

The choice of Relational Database for these projects is PostgreSQL.
Database tables consist of:
- columns: attributes of a row
- row: singular object like structure that houses varius data
Also tables can be linked to each other using a special column called keys, hence the name relational
Each tabe has a primary key that PostgreSQL assigns. A table can reference another table by storing the other's primary key in a column in what is known as 'foreign key'

Seeds are initial sets of data which can be used to populate a database and its tables

Querying data syntax:
SELECT [list of columns] FROM [table_name];

Updating a value:
UPDATE [table_name] SET [column_name] = [new_value];

Deleting a specific rows that match a certain condition
DELETE FROM [table_name] WHERE [conditions];

Adding new row
INSERT INTO [table_name]
( [list of columns] )
VALUES( [list of values] );

To turn the data obtained from PostgreSQL, use a ‘Model’ and ‘Repository’ classes (sometimes called ORM). Model class will hold a record’s data, for example, an attribute for each column. A repository class holds methods to run SQL queries on the database to retrieve, create, update or delete data.

A schema is useful for designing tables and their properties.

Relationship in tables:
- one to many
- many to many

In one-to-many, the many has the foreign key that links to the one table
A JOIN query is used to retrieve data from more than one table. Without using this process it would take twice as many queries and make program/database slow.

SELECT [columns to select] FROM [table name] JOIN [other table name] ON [join condition]

Many-to_Many
- Both table has a relationsip with multiple tables
- A join table is used to connect these two; it links their id to each other

SELECT [columns to select]
FROM [table name]
JOIN [join table] ON [join condition]
JOIN [other table] ON [join condition]
WHERE [condition]

Web Application

It is essential to understand the distinction between client and server.
Client is a software that connects to the server to obtain data or input data in some capacity. The server is essentially another computer that has connection with the database and can run operations that the user has requested from their client machine.

HTTP
Hyper Text Transfer Protocol is a language used to help connect clients to the their specified server on the internet (network). HTTP acts as the roadman creating a path between the client to the server so that they can communicate with each other.
The request and reponse cycle refer to how the client and server interaction procedes.

There is 4 main HTTP Request methods:
- Post: creating/uploading data to the server’s database
- Get: read data, specify what to get
- Put: updating database entries
- Delete: removing database entries

The typical request line includes the method, path and HTTP version and headers provide extra infromation about the request.
The typical response from the server is made up of a status code indicating how the request went and a body which is the content returned.

MVC:
- Model: it models the how data is handled, structured and viewed
- Controller: part of the application that handles user request and provide appropriate responses for the client.
- View: it controls how the response is structured and represented to the user
